Deployment Failures

Use /logs to see the build output. Common causes: syntax error in Dockerfile, missing base image, or network issues during pip install. Fix your code and re-upload.

Check /logs for runtime errors. Make sure all required env variables are set with /env, and all dependencies are in requirements.txt.

Use /disconnect then /connect to re-authorize GitHub access. Then try /github or /update again.

/repos shows empty list

Your GitHub token may have expired. Re-connect using /connect.
